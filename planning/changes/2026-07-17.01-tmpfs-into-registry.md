# Design: move tmpfs into the service-key registry

## Summary

`tmpfs` is handled outside `SERVICE_KEYS` today — validate in
`parsing._validate_tmpfs`, emit in the `--tmpfs` branch of `emit._volume_flags`,
extends-concat in `extends._STRUCTURAL_CONCAT_KEYS`/`_SCALAR_FORM_KEYS` — even
though it is a uniform scalar-or-list flag key. This moves it into the registry
as `"tmpfs": _scalar_or_list("--tmpfs")`, deleting the three hand-rolled halves,
so `tmpfs` gains the single-spec locality (validate + emit + merge in one place)
the registry keys already have. Behavior-preserving.

`volumes` is **not** moved (see Non-goals): its emit needs `project_dir` and its
validation is partly document-level, making it a poor registry fit.

## Motivation

`tmpfs` re-implements abstractions the registry already trusts: its validate is a
plain scalar-or-list-of-strings check, its emit is `--tmpfs <item>` per entry
(no `project_dir`, no document context), and its extends merge is a list concat
(`keys.concat_list` = `as_list(base) + as_list(local)`, identical to
`extends._as_concat_list`). Three modules hand-roll what one `KeySpec` expresses.
This is the same narrow, uniform-shape move `environment` made in
`2026-07-15.17`, licensed as examined-and-in-scope by
`decisions/2026-07-12-reject-structural-key-registry.md` (a single key that
shares one uniform shape, not a structural-key registry).

## Design

**`keys.py`** — add a scalar-or-list factory and the registry entry:

```python
def _validate_string_or_string_list(name, key, value):   # moved from parsing.py, None-guard dropped
    if not isinstance(value, str | list):
        raise UnsupportedComposeError(f"service {name!r}: '{key}' must be a string or list")
    if isinstance(value, list):
        for entry in value:
            if not isinstance(entry, str):
                raise UnsupportedComposeError(f"service {name!r}: '{key}' entry must be a string")

def _scalar_or_list(flag):
    def emit(value):
        items = [value] if isinstance(value, str) else value
        tokens = []
        for item in items:
            tokens += [flag, Expand(value=str(item))]
        return tokens
    return KeySpec(validate=_validate_string_or_string_list, emit=emit, merge=concat_list)
```

Add `"tmpfs": _scalar_or_list("--tmpfs")` to `SERVICE_KEYS`; remove `"tmpfs"`
from `STRUCTURAL_KEYS`. The old validator's `if value is None: return` guard is
dropped: `_reject_null_values` refuses a null `tmpfs` (not in `NULL_ALLOWED_KEYS`)
before the registry loop, so the guard is already dead — exactly as when
`environment` moved.

**`parsing.py`** — delete `_validate_tmpfs`, `_validate_string_or_string_list`
(now in `keys.py`, unused here after `env_file` stopped sharing it), and the
`_validate_tmpfs(name, svc)` call in `_validate_service`. `tmpfs` is now validated
by the existing `SERVICE_KEYS` loop.

**`emit.py`** — remove the `--tmpfs` block from `_volume_flags` (it becomes
volume-only); update its docstring. `tmpfs`'s `--tmpfs` tokens now come from the
`SERVICE_KEYS` loop.

**`extends.py`** — remove `"tmpfs"` from `_STRUCTURAL_CONCAT_KEYS` and
`_SCALAR_FORM_KEYS`; it now merges via `SERVICE_KEYS["tmpfs"].merge` (`concat_list`,
which `as_list`-normalizes a scalar side exactly as `_as_concat_list` did).

**`decisions/2026-07-17-volumes-stays-hand-rolled.md`** — record why `volumes`
is not moved, so the question is not re-opened (per maintainer ruling).

Invariants held:

- **Gate accept-list**: `SUPPORTED_SERVICE_KEYS = set(SERVICE_KEYS) | STRUCTURAL_KEYS`
  — `tmpfs` only switches sides; the union is identical.
- **Emit**: `--tmpfs <item>` tokens are byte-identical. Their *position* in the
  run line shifts (from the `_volume_flags` slot to the `SERVICE_KEYS`-loop slot),
  but `--tmpfs` mounts are independent and order-inert — same as the `-e` position
  shift `environment`'s move noted.
- **Validate**: same `_validate_string_or_string_list` messages; null still
  refused by `_reject_null_values`.
- **Extends**: `concat_list` and `_as_concat_list` both build on `as_list`, so a
  scalar+list / list+list / scalar+scalar merge is identical.

## Non-goals

- **Moving `volumes`** — deliberately out. Its emit (`_volume_flags`/`_mount_flag`)
  needs `project_dir` (relative bind resolution), which `KeySpec.emit(value)`
  cannot supply without widening the signature for all ~30 keys; and its
  reference check (`_validate_volume_references`) is document-level, staying a
  separate pass regardless. A registry `volumes` would be a thin wrapper around
  three still-custom, still-split pieces at the cost of degrading the core `emit`
  interface. Recorded as a decision.
- **Moving `env_file`** — same `project_dir` emit blocker as `volumes`.
- **Widening `KeySpec.emit`** — the whole point of leaving `volumes`/`env_file`
  out is to keep `emit(value) -> tokens` clean.

## Testing

`just test-ci` at 100% coverage, `just lint-ci`, `just check-planning`.

- `tests/test_keys.py`: `SERVICE_KEYS` count +1, `STRUCTURAL_KEYS` −1, union
  unchanged, the two sets still disjoint; `tmpfs` carries `merge=concat_list`.
- `tests/test_emit.py`: the existing `test_tmpfs_string_form`/`test_tmpfs_list_form`
  (tmpfs alone → `flags[4:…]`) still pass (tmpfs alone is the only present
  `SERVICE_KEY`, so it lands at index 4 as before); add a combined case (tmpfs +
  another registry key) confirming the `--tmpfs` tokens are still all present and
  well-formed regardless of order.
- `tests/test_parsing.py`: `tmpfs` string/list accepted, non-string / non-string-
  entry / mapping refused with the same messages; a null `tmpfs` still refused.
- `tests/test_extends.py`: a `tmpfs` extends merge (scalar+list, list+list,
  scalar+scalar) is unchanged; add one if none exists.
- Conformance + integration suites green (no behavior change).

## Risk

- **A `--tmpfs` position assertion in an emit test breaks** (low × low): the
  existing tests use `tmpfs` alone (index 4 preserved); a combined test asserts
  presence, not a fixed index. Caught by `just test-ci`.
- **An extends `tmpfs` merge diverges** (low × med): `concat_list` vs
  `_as_concat_list` are both `as_list`-based — equivalent — and the registry
  branch additionally runs `spec.validate` per side (the drift-close
  `environment` also gained); covered by the extends test above.
- **The moved validator's dropped None-guard changes null handling** (low × low):
  `_reject_null_values` already refuses a null `tmpfs` before the registry loop —
  verified by the null-tmpfs test.
